# Singularity Style Axes

`italic` in the Singularity colorscheme carries three meanings — **foreign** (identifiers from stdlib / builtin), **declaration** (storage / declaration keywords: `def`, `class`, `import`, `let`, `const`, `type`, `interface`), and **organizational** (modules, parameters, comments, doc strings, markup quotes). The three meanings share a struct today but are kept as separately named functions in `style_rules.lua` (`r.foreign` / `r.declaration` / `r.organizational`) so each call site in `color_semantic.lua` documents its intent, and so future divergence (e.g. de-saturating one axis) is cheap. The meanings do not collide visually because the reader disambiguates by token kind. `bold` is reserved for the **definition site** only — `type_definition` and `constructor` — so `class Foo` lights up but `: Foo` reads as a plain reference; `strikethrough` is **deprecated**; `underline` is **readonly / url**. Alternatives considered and rejected: (a) declaration-bold (visually too heavy, plus collided with the definition-site axis); (b) hue-split keywords (control flow orange, declaration cream / slate) — broke the Ubuntu-orange identity that the user wanted preserved; (c) method-vs-free decoration on `func` / `method` — produced noise without aiding reading. Aesthetic reference: rose-pine and catppuccin (italic-heavy, light bold). Heritage: orange `#E95420` from the prior Ubuntu Theme is retained as the primary accent.
